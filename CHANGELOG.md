# CHANGELOG｜詩境版本更新紀錄

本文件整理「詩境」自初版至 v1.4.2 的主要功能演進、資料清理、介面修正與文件調整。

## v1.4.2｜首頁範例按鍵調整

本版本以 v1.4.1 為基底，僅調整首頁範例按鈕，使無 API Key 狀態下也能優先命中本機 `poems.json` 詩庫。

### 本版修正

- 將首頁範例改為本機詩庫確定存在的作者、篇名與詩句片段。
- 更新 `index.html`、`style.css`、`app.js` 的快取版本參數為 v1.4.2。
- 未更動核心查詩流程、Gemini API 流程、收藏、朗讀與雙魂對話邏輯。

## v1.4.1｜修正非必要輸出與格式

本版本為 v1.4.0 後的修正版，集中處理資料輸出、閱讀版面與期末提交文件，作為正式展示候選版本。

### 詩庫輸出清理

- 移除本機詩庫輸出中的 `譯文及注釋`、`譯文`、`直譯`、`韻譯`、`意譯`、`漢譯` 等標題。
- 新增 `注釋`、`註解`、`英譯` 等資料標籤過濾，避免原始資料標籤進入「詩魂自述」或「魂歸何處」。
- 白話釋義區只保留主要中文內容。
- 避免 `others` 欄位中的 `英譯` 被當成創作背景顯示。
- 新增中文優先選取邏輯，自動略過純英文或英文翻譯段落。
- 清理 `poems.json` 中獨立英文翻譯項目。

### 長序文與原詩正文分離

- 將長篇作品的題序從 `content` 移至 `others`，避免序文被當作原詩直排顯示。
- 已處理作品包含《望月有感》《賊退示官吏》《石魚湖上醉歌》《琵琶行》《觀公孫大娘弟子舞劍器行》《燕歌行》。

### 閱讀與版面改善

- 放大原詩直排字級。
- 放大篇名、作者資訊、白話釋義、詩魂自述、對談文字與手機版主要內容。
- 長篇作品維持直排閱讀，原詩白色方框會依最長詩句自動垂直伸縮。
- 長詩欄位過多時保留橫向捲動，兼顧古典直排形式與可讀性。
- 更新前端版本參數，降低 GitHub Pages 或瀏覽器快取造成舊樣式殘留的機率。

### 文件整理

- 精簡發布檔案結構，保留網站核心檔案與期末展示必要文件。
- 重新撰寫 `README.md`、`USER_GUIDE.md`、`docs/FINAL_REPORT.md`、`docs/DEMO_VIDEO_SCRIPT.md` 與 `docs/SOUL.md`。
- 將文件整理為正式專案文件，補齊專案動機、系統架構、使用流程、展示腳本、版本脈絡與反思內容。

## v1.4.0｜校注分離與加入版本參數

- 新增校注／異文自動分離機制。
- `(唯 通：惟)`、`(明年 一作：年年)`、`(顰 一作：蹙)` 等註記不再混入原詩正文。
- 校注內容會獨立顯示，維持原詩閱讀乾淨度。
- 將 `poems.json` 讀取改為避免舊快取，並加入版本參數。
- `index.html` 中的 `app.js` 與 `style.css` 加入版本參數。

## v1.3.0｜API 連接機制與手機版介面優化

- API Key 改存 `sessionStorage`，關閉分頁後清除。
- 新增 API 狀態列：已連線、僅本機、配額受限。
- 加入 429 配額冷卻，避免重複自動測試造成更多錯誤。
- 修正手機版導覽、API 狀態位置、標題遮擋與白框高度問題。
- 移除重複詩藏入口，簡化操作流程。

## v1.2.0｜互動功能擴充

- 新增隨機召喚功能。
- 新增收藏功能，使用 localStorage 保存詩作。
- 新增複製功能。
- 新增瀏覽器內建朗讀功能，不依賴外部 TTS API。
- 新增詩魂問答。
- 新增雙魂對話。

## v1.1.0｜檔案拆分與本機詩庫

- 將原本集中在 `index.html` 的內容拆分為：
  - `index.html`
  - `style.css`
  - `app.js`
  - `poems.json`
- 將詩庫資料移出 HTML，改由 `poems.json` lazy-load。
- 改善 GitHub Pages 部署與後續維護性。

## v1.0.0｜初版MVP

- 建立「通靈問詩」主體互動。
- 支援輸入詩名、作者、片段或意境，透過 Gemini 生成解讀。
- 初步完成詩魂視覺、原詩展示與分析輸出。
